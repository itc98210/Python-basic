🔧 什麼是 SKILL（技能）？
=================

\*\*SKILL（技能）\*\*指的是：  
👉 讓 AI 或系統「能做某件特定事情」的功能模組

你可以把它想成：

| 比喻 | 說明 |
| --- | --- |
| 📱 App | 每個 Skill 就像一個小功能 App |
| 🧠 AI能力模組 | 讓 AI 多一項專長 |
| 🔌 插件 | 插上去就能用的新功能 |

* * *

🧠 SKILL 的核心概念（教學版）
===================

1️⃣ 功能模組化
---------

*   每個 Skill 專門負責一件事
*   例如：
    *   生成圖片
    *   查資料
    *   自動寫報告
    *   操作 Excel

👉 **重點：一個 Skill = 一種能力**

* * *

2️⃣ 可被「觸發」
----------

Skill 不是一直運作，而是：

👉 **透過指令 / 條件 才啟動**

常見觸發方式：

| 類型 | 範例 |
| --- | --- |
| 🗣️ 關鍵字 | 「幫我生成圖片」 |
| ⚙️ 指令 | `/img`、`/search` |
| 🤖 自動判斷 | AI判斷你需要查資料 |
| 📂 檔案事件 | 上傳文件自動分析 |

* * *

3️⃣ 輸入 → 處理 → 輸出
----------------

每個 Skill 都有標準流程：

```
輸入(Input) → 處理(Process) → 輸出(Output)
```

📌 範例（QR Code Skill）

*   輸入：一段文字
*   處理：轉換成QR格式
*   輸出：QR Code圖片

* * *

⚙️ SKILL 結構（技術視角）
=================

以 YAML / AI Agent 常見結構：

```
name: QRCodeGenerator
description: 將文字轉換為QR Code

triggers:
  - keyword: "QR"
  - command: "/qr"

inputs:
  - text: string

outputs:
  - image: png
```

👉 你可以教學生這樣理解：

| 區塊 | 功能 |
| --- | --- |
| name | 技能名稱 |
| description | 做什麼 |
| triggers | 怎麼啟動 |
| inputs | 要什麼資料 |
| outputs | 會產出什麼 |
